# 阿里云 MVP 部署检查清单

## 适用场景

适用于早期 Web 产品在阿里云上快速落地 MVP，尤其是使用轻量应用服务器、OSS、ACR、RDS PostgreSQL Serverless 的组合。

## 推荐资源组合

- 计算：轻量应用服务器或 ECS，运行 Web App、API 和 Worker。
- 镜像仓库：ACR 私有仓库，避免直接依赖 Docker Hub。
- 对象存储：OSS 私有 Bucket，存放生成文件、附件或静态产物。
- 数据库：RDS PostgreSQL Serverless，适合早期访问量不确定的关系型业务。
- 入口：Nginx 或 Caddy 反向代理，后续接 HTTPS 与域名。

## 服务器基础配置

- 使用 Ubuntu LTS 等通用系统镜像。
- 启用 SSH 密钥登录，关闭密码登录。
- 防火墙只开放必要端口，通常是 `22`、`80`、`443`。
- 安装 Docker 与 Docker Compose。
- 配置 Docker 容器日志大小上限，避免系统盘被日志打满。
- 为应用日志配置 logrotate。

## 凭证管理

- 不在仓库提交真实 `.env`、AccessKey、数据库密码、ACR 密码。
- 模板文件只保存变量名和非敏感默认值。
- 使用 `.gitignore` 显式忽略真实环境变量文件。
- RAM 用户必须最小权限，优先只授权到具体 Bucket 或必要云资源。
- 条件允许时，用 RAM 角色或短期凭证替代长期 AccessKey。

## OSS 配置

- Bucket 默认私有。
- 开启阻止公共访问。
- 应用侧只保存 `storage_key`，不要把云厂商完整 URL 写死进业务数据模型。
- 公开访问场景优先通过应用网关、CDN 回源或签名 URL 实现。

## ACR 配置

- 早期可以先使用本地构建并手动推送。
- GitHub 自动构建适合在 Dockerfile、构建脚本和运行时环境变量方案稳定后启用。
- 构建镜像不应依赖生产密钥；生产密钥应在容器运行时注入。

## RDS 配置

- 早期关系型业务优先选择 RDS PostgreSQL。
- Serverless 适合访问量不确定的 MVP，最小 RCU 可低配起步。
- 最大 RCU 不宜一开始设置过高，避免异常流量导致费用不可控。
- 优先使用内网地址访问数据库。
- 白名单只放行服务器内网 IP 或最小必要网段。
- 如果轻量应用服务器无法访问 RDS VPC，再考虑公网地址，并限制公网白名单到服务器公网 IP。
- 业务应创建独立数据库和应用账号，不长期使用默认库或高权限账号。

## 验证顺序

1. 在服务器检查 DNS 是否能解析云资源内网地址。
2. 用 `nc` 或等价工具测试目标端口是否连通。
3. 用官方客户端验证登录和最小查询。
4. 再接入应用配置和部署流程。

## 常见问题

- RDS 地址能解析但端口超时：优先检查白名单、VPC 互通和安全组。
- Docker Hub 拉取超时：优先使用 ACR，或配置可信镜像源。
- ACR 自动构建失败：确认仓库内已有 Dockerfile、构建上下文正确，且构建阶段不依赖运行时密钥。
- `.env` 误写入模板：立即清空模板中的真实值，并轮换已经暴露的密钥。
