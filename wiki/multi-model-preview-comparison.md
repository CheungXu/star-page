# 多模型结果预览对比设计

当一次请求并行产出多个模型的页面结果时，预览区要让用户"快速并排对比、又能聚焦细看单个"。记录可复用的布局与交互原则。

## 复用单结果预览方案

每个对比单元仍是一个"真实浏览器视窗"：固定桌面视口（如 1200px）渲染，再按单元宽度 `scale = 单元可用宽 / 视口宽` 缩放（见 `generated-page-preview-design.md`）。关键点：缩放按"每个单元自己的舞台宽高"计算，而不是整个预览区，所以把缩放逻辑封装进独立的预览单元组件，各单元各算各的。

## 自适应网格 + 聚焦

- 1 个结果：占满。
- 2 个：并排两列。
- 3-4 个：两列自动换行成 2x2。
- 更多：两列 + 纵向滚动（或退化为标签切换）。
- 列数用 `repeat(min(N,2), minmax(0,1fr))` 之类按 N 计算，天然支持扩展。
- 提供单元"聚焦"：点聚焦后网格只渲染该单元（单列占满），顶部给"返回对比"出口；适合细看某个模型结果。

## 给对比腾空间

并排对比时，半屏预览区里每个单元会很小。可在多模型时切换布局：让左侧会话栏收窄、右侧预览栏加宽（如对比模式下 `0.82fr : 1.45fr`），单结果时回到 1:1。窄屏回退为单列竖排。

## 每个单元的信息与动作

- 头部：模型标签 + 状态点/状态胶囊（思考中/创建中/已生成/失败）。
- 完成态：浏览器视窗预览 + 打开页面 + 复制链接（各结果独立可分享）+ "以此结果继续"（基于该节点续写/分支）。
- 进行态：骨架屏 + 当前状态文案；失败态：单元内显示错误，不影响其它模型单元。

## 经验

- 不同模型耗时不同：并排时各单元独立从"骨架屏 → 完成预览"切换，先完成的先展示，不要等全部完成再统一渲染。
- 每个结果都是独立可分享节点，复制/打开/继续都以"该节点的 page_url / page_id"为准，便于用户挑选、分享、分支。
