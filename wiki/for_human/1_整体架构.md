# 1_整体架构

这份文档用最简单的话说明当前项目长什么样、每个云资源负责什么。

## 一句话说明

这是一个“用自然语言生成 HTML 页面”的网站。

用户在主站输入需求，系统调用大模型生成 HTML，把页面文件放到 OSS，页面信息和权限放到数据库，最后给用户一个可访问链接。

## 当前整体结构

```text
用户浏览器
  -> Nginx
  -> Next.js 前端
  -> FastAPI 后端
  -> RDS PostgreSQL
  -> OSS
  -> LLM 大模型
```

部署和镜像：

```text
GitHub
  -> 服务器拉代码
  -> Docker 构建镜像
  -> ACR 保存镜像
  -> 服务器运行容器
```

## 每一块负责什么

## 1. 服务器

当前使用阿里云轻量应用服务器。

它负责运行：

- Next.js 前端
- FastAPI API 服务
- 当前内置在 FastAPI 中的 AI 生成流程
- Nginx 反向代理
- Docker 容器

它不应该长期保存用户生成的 HTML 文件。

## 2. Nginx

Nginx 是网站入口。

以后用户访问域名时，请求会先到 Nginx，再转发给应用容器。

当前还没有正式域名和 HTTPS，后续需要补：

- 域名解析
- HTTPS 证书
- 正式反向代理配置

## 3. Docker

应用以后用 Docker 跑。

好处：

- 部署更稳定
- 服务器环境更干净
- 以后迁移服务器更容易

当前已有 `code/frontend/` 和 `code/backend/` 两个应用目录，并新增了 `code/docker-compose.yml` 编排模板。

## 4. ACR 镜像仓库

ACR 是阿里云镜像仓库。

用途：

- 保存 Docker 镜像
- 避免直接依赖 Docker Hub
- 后续服务器可以从 ACR 拉镜像部署

当前镜像地址：

```text
crpi-6w1a91eyh3y1vcd9.cn-guangzhou.personal.cr.aliyuncs.com/stars-page-demo/stars-page-demo
```

## 5. OSS 对象存储

OSS 用来保存用户生成的 HTML 文件。

例如未来可能是：

```text
pages/{page_id}/versions/{version_id}/index.html
```

当前 Bucket：

```text
stars-page-demo
```

Bucket 保持私有，不直接公开。

公开访问以后再通过：

- 应用鉴权后返回
- CDN 回源
- 签名 URL
- 页面访问网关

## 6. RDS PostgreSQL

RDS 保存业务数据。

例如：

- 用户
- 页面
- 页面版本
- 页面权限
- AI 生成任务

当前使用 RDS PostgreSQL Serverless。

服务器已经能通过内网连接 RDS。

当前业务数据库为：

```text
stars_page
```

不要长期使用默认 `postgres` 数据库放业务表。

## 7. LLM 大模型

LLM 用来根据用户自然语言生成 HTML。

当前已接入阿里云百炼 Qwen。

配置方式：

```text
LLM_PROVIDER=qwen
LLM_PROTOCOL=openai
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen3.7-max
```

代码里已经抽象了两种主流协议：

- OpenAI-compatible
- Anthropic-compatible

以后换模型时，优先只改配置，不改业务代码。

## 8. GitHub

GitHub 保存代码。

服务器已经配置 SSH key，可以推送代码到：

```text
git@github.com:CheungXu/star-page.git
```

真实密钥不会提交到 GitHub。
