# 0_初始化配置

这份文档写给人类操作。目标是：新开一台服务器后，照着做，把项目运行需要的基础东西配好。

## 1. 准备服务器

推荐配置：

- 系统：Ubuntu LTS
- CPU：2 vCPU 起步
- 内存：4 GiB 起步
- 系统盘：50 GiB 起步
- 端口：只开放 `22`、`80`、`443`

先确认能 SSH 登录服务器。

## 2. 配 SSH 登录

本地保存服务器私钥，例如：

```bash
chmod 600 ~/.ssh/你的服务器私钥.pem
```

配置本机 `~/.ssh/config`：

```text
Host star-page-aliyun
  HostName 服务器公网IP
  User root
  Port 22
  IdentityFile ~/.ssh/你的服务器私钥.pem
```

测试：

```bash
ssh star-page-aliyun "uname -a"
```

确认密钥登录可用后，服务器上关闭 SSH 密码登录。

## 3. 安装服务器基础环境

在服务器执行：

```bash
apt-get update
apt-get install -y docker.io docker-compose-v2 nginx ufw logrotate postgresql-client
systemctl enable --now docker nginx
```

防火墙：

```bash
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
```

## 4. 配 Docker 日志

创建 `/etc/docker/daemon.json`：

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "live-restore": true
}
```

重启：

```bash
systemctl restart docker
```

## 5. 配 GitHub SSH

在服务器生成 GitHub key：

```bash
ssh-keygen -t ed25519 -C "star-page-server-github" -f ~/.ssh/star_page_github_ed25519 -N ""
```

配置 `~/.ssh/config`：

```text
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/star_page_github_ed25519
  IdentitiesOnly yes
```

查看公钥：

```bash
cat ~/.ssh/star_page_github_ed25519.pub
```

把公钥加到 GitHub 仓库 Deploy keys，勾选 `Allow write access`：

```text
https://github.com/CheungXu/star-page/settings/keys
```

测试：

```bash
ssh -T git@github.com
```

## 6. 配 ACR 镜像仓库

在阿里云 ACR 创建：

- 地域：华南 3（广州）
- 命名空间：`stars-page`
- 仓库：`stars-page`
- 类型：私有

服务器登录 ACR：

```bash
docker login --username=<阿里云账号或RAM用户名> crpi-6w1a91eyh3y1vcd9.cn-guangzhou.personal.cr.aliyuncs.com
```

不要把 ACR 密码写进仓库或聊天。

## 7. 配 OSS

在阿里云 OSS 创建 Bucket：

- Bucket：`stars-page-demo`
- 地域：华南 3（广州）
- 权限：私有
- 阻止公共访问：开启

创建 RAM 用户或角色，只给这个 Bucket 的最小权限。

真实配置写到 `config/oss.env`：

```text
OBJECT_STORAGE_PROVIDER=aliyun
OBJECT_STORAGE_BUCKET=stars-page-demo
OBJECT_STORAGE_REGION=cn-guangzhou
OBJECT_STORAGE_ENDPOINT=oss-cn-guangzhou.aliyuncs.com
OBJECT_STORAGE_ACCESS_KEY_ID=
OBJECT_STORAGE_ACCESS_KEY_SECRET=
```

## 8. 配 RDS 数据库

在阿里云 RDS 创建：

- 产品：RDS PostgreSQL
- 付费：Serverless
- 地域：华南 3（广州）
- 最小 RCU：`0.5`
- 最大 RCU：先用 `2` 或 `4`
- 网络：优先内网

白名单放行服务器内网 IP，例如：

```text
172.19.39.58
```

真实配置写到 `config/db.env`：

```text
PGHOST=RDS内网地址
PGPORT=5432
PGDATABASE=postgres
PGUSER=应用账号
PGPASSWORD=
```

测试：

```bash
bash script/check_postgres_connection.sh
```

## 9. 配 LLM

阿里云百炼 Qwen 使用 OpenAI-compatible 格式。

真实配置写到 `config/llm.env`：

```text
LLM_PROVIDER=qwen
LLM_PROTOCOL=openai
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen3.7-max
LLM_API_KEY=
LLM_ENABLE_THINKING=true
LLM_TIMEOUT_MS=60000
```

`LLM_API_KEY` 填 DashScope / 百炼 API Key。

## 9.1 配费用对账凭据（财务后台用，可选）

财务后台「供应商真实余额对账 / 成本账单」需要各云厂商的**财务只读**凭据，和业务用的 key 分开、最小权限。这两个文件由 systemd 以可选方式加载（缺失不影响启动）。

阿里云费用查询：新建一个**专用 RAM 用户**，授予系统策略 `AliyunBSSReadOnlyAccess`（费用中心只读，含查余额 + 查账单）。配置写到 `config/aliyun.env`：

```text
ALIYUN_BILLING_ACCESS_KEY_ID=
ALIYUN_BILLING_ACCESS_KEY_SECRET=
```

火山引擎（豆包）余额：需要**火山账号 IAM 的 AK/SK**（注意：ARK 推理 key 不能查余额），授予财务只读权限。配置写到 `config/huoshan.env`：

```text
VOLC_ACCESSKEY=
VOLC_SECRETKEY=
```

改完执行 `sudo systemctl restart star-page-backend` 生效。

## 10. 最重要的安全规则

- 不要把任何真实 key 发到聊天里。
- 不要把任何真实 key 写进 README、wiki、代码或镜像。
- 真实密钥只放 `config/*.env`。
- `config/*.env` 必须被 Git 忽略。
- 如果 key 暴露过，立刻删除并重新生成。

检查真实 env 是否被忽略：

```bash
git check-ignore -v config/oss.env config/db.env config/llm.env config/aliyun.env config/huoshan.env
```
