# 星页计费系统实施记录（20260614）

> 面向项目管理的实施记录，隐去具体代码细节。设计方案见同目录 `billing-system-plan.md`。

## 一、本次目标

为星页上线以积分为单位的完整计费体系：匿名试用 → 注册合并的用户额度体系、按实际消耗 × 模型倍率扣费、积分充值购买页（支付先 mock）、以及基于复式记账的管理员财务后台。

## 二、已交付内容

1. **数据与会计内核**
   - 数据库迁移 `011_billing.sql`：积分钱包、积分流水、充值订单、套餐、复式记账总账三表，并为用户增补匿名/归并字段、新增匿名访客表；启动 seed 会计科目与 mock 套餐。
   - 计费服务：定价（向上取整、模型倍率、最小 1 积分兜底）、钱包（赠送/充值入账、生成结算，扣减顺序先赠送后充值）、复式过账（借贷必平、按事件幂等）。
   - 定价纯逻辑单测全部通过。

2. **匿名 → 注册闭环**
   - 未登录访客自动获得签名 cookie 身份，免费体验 2 次；用尽或复制分享链接时引导登录。
   - 手机验证码登录时把匿名期的会话/页面归并到正式账号，并首次赠送 1000 积分（幂等）。

3. **生成链路计费**
   - 生成前置校验：匿名查免费次数 + 模型白名单 + 单次模型数上限；登录查积分余额，不足返回 402 引导充值。
   - 生成成功后按页面版本幂等结算：登录用户扣积分并确认收入、记算力成本；匿名仅记成本（计入试用获客成本）。

4. **用户侧**
   - 顶部展示积分余额 / 剩余免费次数；低余额与匿名拦截引导。
   - 充值购买页：mock 套餐卡（折扣/赠送标注）+ mock 支付闭环 + 积分流水。

5. **财务后台**
   - 管理员（手机白名单）专属：财务总览（充值现金、确认收入、算力成本、毛利及毛利率、赠送已发/未用负债、预收账款、应付 LLM、科目余额）、积分流水、记账凭证、用户对账。

## 三、安全加固要点

- **匿名防滥用**：HMAC 签名 cookie 只认自家签发；按 IP 每日限制签发匿名 id 数与免费生成次数；匿名仅可选次优模型且单次 ≤2 个，后端强校验防绕过前端。
- **充值资金安全**：价格服务端权威（客户端只传套餐 key）；建单/回调按唯一约束 + 原子状态流转幂等；mock 支付仅非生产环境开放；真实回调预留验签 + 金额比对 + 订单归属校验。

## 四、上线验证

- 数据库迁移成功执行。
- 后端重启后冒烟：套餐、账户、模型（含匿名可用标记）接口正常；管理员与充值接口未登录均 401。
- 匿名围栏端到端：连续生成第 1、2 次成功并签发 cookie、免费次数递减，第 3 次返回 402 引导登录；匿名选高价模型或超过 2 个模型均被后端拒绝。
- 前端生产构建通过并重启，`/`、`/pricing`、`/admin` 均 200，静态资源 200。
