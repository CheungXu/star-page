# 侧边栏账号区交互与排版精修记录

## 背景

根据线上预览反馈，侧边栏折叠态的底部用户缩略图标仅用于登录入口，用户期望它也能像顶部品牌按钮一样展开侧边栏；同时展开态侧边栏宽度偏窄，底部积分与账号区域信息拥挤，需要进一步提升视觉层级与可点击性。

## 本次交付

- 折叠态侧边栏支持更多展开入口：
  - 点击底部用户头像可展开侧边栏。
  - 点击折叠侧边栏空白处也可展开。
  - 点击按钮、链接、输入框、历史项等交互元素时仍执行原操作，避免空白展开逻辑抢事件。
- 展开态侧边栏宽度从 `208px` 调整为 `252px`，首页、工作区、多模型对比布局及中等屏宽 sticky 状态保持一致。
- 底部账号区重排为两段式：
  - 上方积分/免费体验卡片，使用标签、大数字、折算金额/提示文案分层展示。
  - 下方操作按钮区，`充值` 作为主操作使用实心渐变按钮，`财务后台` 作为次级入口。
  - 账号行独立展示头像、登录状态和退出按钮，头像加入柔和渐变、描边和阴影。

## 设计取舍

- 折叠态整条侧边栏加 `cursor: pointer`，用视觉反馈提示“可展开”，但事件处理仍排除交互元素，保证可用性。
- 余额区域不再使用单行 chip。积分是财务/转化相关核心信息，采用卡片化和大数字能更快建立信息锚点。
- `充值` 是用户最常用的付费转化入口，视觉权重高于 `财务后台`；管理员入口保留但弱化。

## 上线与验证

本次仅改前端：

```bash
cd /root/star-page/code/frontend
npm run build
systemctl restart star-page-frontend.service
```

验证结果：

- `http://127.0.0.1:3000/` 返回 `200`。
- 首页引用的 `/_next/static/...css` 返回 `200 text/css`。
- Nginx 公网首页 `https://stars-page.com/` 返回 `200`。

## 后续注意

侧边栏宽度、工作区三栏比例和多模型对比栏宽强相关。后续继续调宽侧边栏时，需要同步检查：

- 首页 Hero 是否仍视觉居中。
- 工作区单模型/多模型预览宽度是否被挤压。
- `max-width: 1100px` 附近的响应式布局是否仍能正常从三栏切到双栏。
