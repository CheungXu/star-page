# 历史记录管理功能实现记录

## 背景

侧边栏历史记录需要支持单条删除、收藏、收藏筛选和关键词检索。当前历史记录已从页面维度升级为会话维度，因此本次以 `conversations` 表作为功能主体。

## 实现范围

- 数据库新增 `conversations.is_favorite` 字段，删除继续使用既有 `deleted_at` 软删除字段。
- 历史列表接口 `GET /api/conversations` 支持 `favorite_only` 和 `q` 查询参数。
- 新增 `PATCH /api/conversations/{conversation_id}` 更新收藏状态。
- 新增 `DELETE /api/conversations/{conversation_id}` 软删除会话。
- 前端侧边栏新增收藏入口、检索输入框、单条收藏按钮和删除按钮。

## 交互说明

- 删除历史只影响历史列表展示，不物理删除页面、批次和生成产物。
- 收藏筛选与关键词检索可叠加：在收藏视图下检索时只返回匹配的收藏记录。
- 删除当前正在查看的会话后，前端会清理当前本地会话并回到新对话状态。

## 验证要点

- 新建会话后能在全部历史中看到记录。
- 收藏记录后，切换到收藏视图只显示收藏记录。
- 在全部历史和收藏视图中输入关键词，列表只显示匹配记录。
- 删除记录后刷新仍不再显示。
