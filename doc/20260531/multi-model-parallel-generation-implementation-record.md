# 多模型并行生成实施记录（Scheme A）

对应规划见同目录 `multi-model-parallel-generation-plan.md`。本文记录实际落地情况、与规划的增量决策、过程中发现并修复的问题，以及线上验证与当前限制。

## 已交付能力

- 新对话可多选模型并行生成；首页输入卡下方按"模型目录"动态渲染多选（默认只勾 qwen，doubao 已注册可选）。
- 一次请求 = 一个"批次"，批次内每个模型一个独立可分享的页面节点，N 路独立 SSE 并行推进，先完成先展示。
- 右侧预览区并排对比各模型结果，单元可"聚焦"放大、各自打开/复制链接；中等宽度自动改为上下堆叠全宽。
- 多轮续写：在工作区直接发送即"在本会话内续写"。
- 已接入 qwen（通义千问 3.7 Max）与 doubao（豆包 Seed 2.0 Pro，火山方舟 ARK，`reasoning_effort=medium`），均为 OpenAI 兼容。

## 与规划的增量决策

1. 续写有两种语义（实现并验证）：
   - 未选具体节点 → 并行续写：每个模型各自接住"自己那条链"的上一版结果继续，同会话内各自成链。
   - 在某结果卡片点「以此结果继续」→ 分支：所有选中模型从该节点派生。
   - 首页提交才新建会话；工作区提交一律续写。
2. 续写改为"内容感知"：消息结构 = system(通用 HTML 规则) + user(历次全部指令清单) + assistant(仅最近一版完整 HTML) + user(本轮新指令，附"未提到的部分保持不变")。即"指令全带、答案只裁成最近一版"——HTML 是完整快照，更早版本冗余，避免上下文爆炸；想回更早版本走生成树从旧节点分支。
3. 模型目录与参数分层落地为可提交的 `config/llm.models.json`（defaults + 每模型 params/extra_body 三层覆盖），密钥放 gitignored 的 `config/llm.env`，密钥缺失的模型自动不可选；qwen 的密钥兼容回退到旧 `LLM_API_KEY`。
4. 单次输出上限 `max_tokens` 由 8192 上调到 65536（defaults），避免复杂页面/整页重写被截断。

## 过程中发现并修复的问题

- 状态胶囊被撑成竖条：模型状态词 `creating` 直接当 CSS 类名，与旧单模型遗留的全局 `.creating`（`min-height:240px`）撞名。修复：状态类名命名空间化（`is-${status}`）。
- 并行续写血缘：让每个新节点的父 = 同模型上一轮节点（后端按 `model_key` 找会话内最新节点）。数据级校验通过（qwen→qwen、doubao→doubao 各自成链）。
- 中等宽度（约 860~1100px）布局塌陷：对比模式媒体查询把三栏降为两栏却没重排预览栏，预览被挤进 72px 侧栏列、标题压成竖排。修复：媒体查询里显式把预览栏放到第 2 列、侧栏跨两行、预览网格单列竖排。
- 侧边栏历史项被拉伸：`.history-list` 网格在高侧栏里、项目少时被默认 `align-content: stretch` 等高拉满，标题与日期被上下拉开。修复：`align-content: start` + `grid-auto-rows: max-content`。

（上述前端通用经验已抽到 `wiki/frontend-css-grid-and-class-pitfalls.md`。）

## 线上验证

- `GET /api/models` 返回 qwen(默认)+doubao(可用)；2 模型并行生成两路均成功产出独立页面，批次状态聚合为 succeeded，`/p/{page_id}` 正常渲染。
- 多轮：第 2 轮"换绿色系+加会员区块"、第 3 轮"只把年度会员价格改成 ¥99，其它不变"——模型思考明确引用上一版页面结构后改价，续写内容感知生效。
- 前端生产构建 + systemd 重启后线上 `http://8.138.118.232/` 实测通过。

## 当前限制与后续

- 合并（多节点→新会话根）仅预留 schema（`batch.kind=merge_seed` + `source_page_ids`），未做端到端。
- N 路 SSE 受 HTTP/1.1 单源约 6 连接限制，默认 2 个无碍；模型大量增加后需升级"批次多路复用 SSE"或轮询。
- 续写把"上一版 HTML"作为上下文，若该版曾被 `max_tokens` 截断，模型需重建整页；已上调 max_tokens 缓解。
- 历史/会话恢复与登录仍沿用默认测试用户，待真实登录接入后切换。
