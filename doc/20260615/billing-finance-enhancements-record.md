# 计费财务后台增强实施记录（预付费改造 + 供应商对账）

承接 `20260614` 计费系统上线后，本轮针对**管理员财务后台**与**真实成本核算**做了一组增强。面向项目管理视角，技术细节见 `code/backend/README.md` 与 `wiki/`。

## 一、背景与目标

计费系统上线后，财务口径上有几处需要更贴近真实经营：

1. 财务总览信息堆在一起，看不清「不含赠送的真实付费状况」与「含赠送的账面状况」。
2. 模型倍率只能改配置文件、需重启，缺少后台维护入口。
3. **供应商是预付费**（先给云账户充值、调用即扣），原先按「应付账款」记账不符合实际资金状况。
4. 想知道云账户**真实余额**、以及账户里**不只 LLM、还有服务器等成本**，需要计入并对账。

## 二、交付内容

### 1. 财务总览三段式 + 营业利润

- **付费业务（不含赠送）**：累计充值现金 / 付费确认收入 / 付费算力成本 / 付费毛利。
- **赠送台账**：赠送已发放 / 未用负债 / 已核销收入 / 赠送+试用成本。
- **含赠送合计**：综合收入 / 成本 / 毛利。
- **营业利润**：综合毛利 − 基础设施成本（见第 4 点）。
- 收入与成本按消费流水的 paid/gift 占比拆分；积分流水类型全部中文化。

### 2. 模型倍率配置页（可视化、即时生效）

- 后台新增「模型倍率」Tab，展示每个模型的厂商、可用状态、**成本基准（输入/输出每百万 tokens 单价）**、当前倍率与是否自定义。
- 保存即写回 `config/billing.json` 并清缓存**即时生效**，无需重启（其余配置如管理员手机号仍需重启）。

### 3. 预付费记账模型改造（应付 → 预付资产）

- 把原「应付账款-LLM供应商(负债 2101)」重分类为「**预付账款-云/LLM供应商(资产 1102)**」，历史凭证平滑迁移（迁移 `012`）。
- 每次调用消费改为**冲减预付资产**（借 6001 算力成本 / 贷 1102），符合"现金已提前充值"的实际。
- 新增管理员「记一笔云账户充值」入口（借 1102 / 贷 1001 现金），使 1102 余额＝云账户账面剩余。

### 4. 供应商真实余额对账 + 账单成本核算

- **真实余额**：后台直连云厂商财务接口拉账户余额做对账（带 5 分钟缓存 + 手动刷新）。
  - 阿里云：BSS `QueryAccountBalance`，用**专用费用 AccessKey**（`config/aliyun.env`，最小权限 `AliyunBSSReadOnlyAccess`）。
  - 火山（豆包）：`QueryBalanceAcct`，用火山账号 IAM AK/SK（`config/huoshan.env`）。已实测拉到真实余额。
- **服务器等基础设施成本计入**（按"自动区分 + 期间费用"口径）：
  - 新增「成本账单」Tab，按账期拉阿里云账单总览（`QueryBillOverview`），按关键词自动拆「百炼 LLM（已按次计入，不重复）」与「服务器等基础设施」。
  - 一键把基础设施部分按账期**幂等入账**（借 6002 基础设施成本 / 贷 1102），并体现到「营业利润」。
- **百炼成本偏差对账**：把该账期内经百炼平台计费的模型（qwen 及在百炼托管的 deepseek/glm/kimi/minimax；doubao 走火山不计入）按次估算 COGS 与账单百炼实际金额对比，给出偏差额/偏差率，用于校准倍率成本基准、发现免费额度红利。

## 三、成本口径决策（重要）

- **采用「折扣后应付 pretax_amount」而非「现金支付 payment_amount」**：预付费下现金在充值时已支付、账单期现金支付≈0，应付口径才等于真实资源消耗（＝对预付资产的冲减）。
- 账单同时展示付款拆解（原价 / 应付 / 代金券补贴 / 储值卡抵扣 / 现金支付），其中阿里云免费代金券抵扣部分明确标为"补贴"（未花钱）。
- 偏差对账账期按 **UTC+8 自然月**与阿里云出账周期对齐，月初月末可能有少量跨期误差。

## 四、上线验证

- 迁移 `012/013` 已执行；前后端已重新构建并经 systemd 重启。
- 火山真实余额实测可拉取；阿里云费用 key 配好后余额/账单均可正常拉取，百炼产品被正确识别。
- 基础设施入账与偏差对账逻辑在真实数据下验证（含幂等：同账期重复入账自动跳过）。

## 五、后续可迭代

- 百炼若改为预付资源包计费，可接 `QueryResourcePackageInstances` 展示包余量。
- 偏差对账可做成每月例行报表/告警（偏差率超阈值提醒校准倍率）。
- 其余厂商（DeepSeek/智谱/Kimi/MiniMax 自有账号）按需新增余额 provider。
