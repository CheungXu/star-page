# 数据报告页专项制作指南

为"数据/结论汇报"场景生成结构清晰、图表得当、专业可信的报告页。

## 信息架构（推荐模块顺序）

1. 报告抬头：标题、副标题、时间范围/版本、作者/团队。
2. 摘要/核心结论：先给结论（TL;DR），3-5 条关键发现。
3. 关键指标卡（KPI）：一排数字卡，含指标名、数值、同比/环比变化（升降用颜色与箭头）。
4. 分主题章节：每章「小标题 + 图表 + 文字解读」，图文并排或上下结构。
5. 明细数据：必要时用表格，支持斑马纹、对齐数字、突出关键行。
6. 结论与建议：行动建议、下一步计划。
7. 附录/数据说明：口径、来源、限制。

## 图表表达

- 数据写死在页面内；推荐用可信 CDN 的轻量图表库（如 cdn.jsdelivr.net 上的 Chart.js）或纯 SVG/CSS 绘制。
- 图表选型：趋势用折线、对比用柱状、占比用饼/环、构成用堆叠；不要为炫技滥用复杂图。
- 每个图表配一句"它说明了什么"的解读，避免只放图不给结论。

## 视觉与排版

- 专业克制的配色：中性背景 + 一主色 + 升/降语义色（绿/红）；信息密度可比落地页高。
- 用卡片、分隔与栅格组织模块；数字用等宽或表格数字对齐。
- 标题层级与留白清晰，便于快速扫读结论。

## 交互（克制）

- KPI 数字可计数器动画；图表入场可渐显；长报告可加锚点目录/回到顶部。
- 表格数据量大时可前端筛选/排序（纯前端，不发请求）。

## 易错点

- 先结论后细节，避免让读者自己从图里找答案。
- 同比/环比要标明正负与口径；颜色语义统一（升=绿/降=红或按业务约定）。
- 移动端图表与表格要可横向滚动或自适应，不被截断。
